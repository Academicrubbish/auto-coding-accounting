### 问题2: 统计页面无数据显示（已修复）✅

**发现时间**: 2026-03-06
**发现方式**: 测试验证时发现统计页面显示为空数据

**问题描述**:
- 首页正常显示：总资产¥-150.00，本月支出¥150.00，3条交易记录
- 统计页面显示：收入¥0.00，支出¥0.00，结余¥0.00，暂无数据

**根本原因**:
云函数使用 `command.gte(startDate).and(command.lte(endDate))` 查询Date对象时存在时区问题，导致查询结果为空

**修复位置**: [uniCloud-aliyun/cloudfunctions/statistics/index.js:61-75](uniCloud-aliyun/cloudfunctions/statistics/index.js#L61-L75)

**修复前**:
```javascript
const startDate = new Date(year, month - 1, 1);
const endDate = new Date(year, month, 0, 23, 59, 59);

const res = await collection.where({
  user_id: openid,
  transaction_date: command.gte(startDate).and(command.lte(endDate))
}).get();
```

**修复后**:
```javascript
// 使用字符串格式避免时区问题
const startDate = `${year}-${String(month).padStart(2, '0')}-01`;
const endDate = `${year}-${String(month).padStart(2, '0')}-31`;

// 先查询所有交易，在内存中过滤
const res = await collection.where({ user_id: openid }).get();

const transactions = res.data.filter(t => {
  const date = new Date(t.transaction_date);
  const dateStr = `${date.getFullYear()}-${String(date.getMonth() + 1).padStart(2, '0')}-${String(date.getDate()).padStart(2, '0')}`;
  return dateStr >= startDate && dateStr <= endDate;
});
```

**状态**: ✅ 已修复 - 云函数已上传部署

**文件**: [pages/record/index.vue](pages/record/index.vue)

**问题**: 编辑交易时，表单数据未完全回填

**建议**:
```javascript
// 在 onMounted 中添加编辑模式数据加载逻辑
const transactionId = ref('')

onMounted(async () => {
  const pages = getCurrentPages()
  const currentPage = pages[pages.length - 1]
  const options = currentPage.options

  if (options.id) {
    transactionId.value = options.id
    await loadTransactionData(options.id)
  }
})

async function loadTransactionData(id) {
  const res = await uniCloud.callFunction({
    name: 'transaction',
    data: { action: 'getById', id }
  })

  if (res.result.code === 0) {
    const tx = res.result.data
    amount.value = tx.amount
    selectedCategory.value = tx.category_id
    selectedAccount.value = tx.account_id
    transactionDate.value = tx.transaction_date
    remark.value = tx.remark
  }
}
```

## 🛠️ 已修复问题总结

### 问题 1: 负数余额显示错误 (M6.1)

**症状**: 统计页面显示 ¥62.00 而不是 ¥-62.00

**根本原因**: 前端使用了 `Math.abs(monthBalance)` 导致所有余额显示为正数

**修复**:
- 文件: `pages/stats/index.vue:30`
- 修改: 移除 `Math.abs()`，保留条件样式类
- 影响: 负余额现在正确显示为红色，正余额显示为绿色

**状态**: ✅ 已修复并验证

---

### 问题 2: 统计页面数据为空 (M6.1)

**症状**: 统计页面查询返回空结果，首页却有数据显示

**根本原因**: uniCloud 云函数使用 Date 对象进行查询时存在时区问题

**修复**:
- 文件: `uniCloud-aliyun/cloudfunctions/statistics/index.js:61-75`
- 修改: 改用字符串格式日期，在应用内存中进行过滤

**状态**: ✅ 已修复并验证

---

### 问题 3: 编辑模式数据未回填 (M5.2)

**症状**: 从详情页点击编辑后，记账页表单为空

**根本原因**:
1. `loadTransactionForEdit` 函数定义在 `onMounted` 之后（const 赋值不支持提升）
2. `handleSubmit` 只支持 'create' 操作

**修复**:
- 文件: `pages/record/index.vue`
- 修改:
  1. 添加 `isEditMode` 和 `editTransactionId` 状态变量
  2. 移动 `loadTransactionForEdit` 函数定义到 `onMounted` 之前
  3. 修改 `handleSubmit` 支持根据 `isEditMode` 选择 'create' 或 'update' 操作
  4. `onMounted` 改为 async，确保基础数据加载后再加载编辑数据

**状态**: ✅ 已修复，待重新编译后验证

---

### 问题 4: 系统分类保护逻辑错误 (M8.4)

**症状**: 点击系统分类后无明显响应

**根本原因**:
1. Badge 显示逻辑错误：使用 `create_by !== ''` 判断
2. 保护检查逻辑错误：使用 `create_by === ''` 判断
3. 实际系统分类的 `create_by` 值为 `'common'`

**修复**:
- 文件: `pages/category/index.vue:38,121` + `uniCloud-aliyun/cloudfunctions/category/index.js:186`
- 修改:
  1. Badge 显示改为 `create_by === 'common'`
  2. 保护检查改为 `create_by === 'common'`
  3. 同步更新云函数注释

**状态**: ✅ 已修复，待重新编译后验证
